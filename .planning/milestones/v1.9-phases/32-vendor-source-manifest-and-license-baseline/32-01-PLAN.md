<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TSV registry -> verifier shell process | Checked-in text controls verifier inputs and must not become executable shell. |
| Local verifier -> upstream Git remotes | `git ls-remote` observes mutable remote refs controlled outside this repository. |
| Metadata -> maintainer/legal interpretation | SPDX and provenance fields can be mistaken for legal review or runtime support. |
| Vendor caution docs -> future implementation scope | Non-free/network-plugin notes can accidentally become ingestion or integration scope. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-01 | Tampering | Upstream selected tags | mitigate | `verify_forks.sh` compares `refs/tags/<tag>` and peeled commit SHAs against checked-in `tag_ref_sha` and `peeled_commit_sha`; mismatch exits non-zero. |
| T-32-02 | Elevation of Privilege | `refresh_command` and TSV fields | mitigate | Verifier never executes TSV command text, forbids `eval`/shell command construction, validates fields before network calls, and quotes all `git ls-remote` arguments. |
| T-32-03 | Repudiation/Tampering | Default branch observations | mitigate | Branch heads are stored and reported as drift-only observations; docs and verifier warnings distinguish them from canonical release-pin acceptance. |
| T-32-04 | Repudiation | License/provenance metadata | mitigate | Registry separates `spdx_identifier`, `observed_license_id`, `license_source`, `provenance_notes`, `caution_flags`, and `caution_notes`, and repeats `metadata-only-not-legal-review`. |
| T-32-05 | Information Disclosure/Supply Chain | Non-free/network plugin cautions | mitigate | Bambu and Orca caution fields and docs state `do-not-ingest-plugin-code-or-credentials`; no source trees, plugin code, credentials, or network integrations are added. |
| T-32-06 | Denial of Service | Upstream Git network availability | accept | Verification may fail if GitHub is unreachable; failure is visible and does not alter checked-in metadata or local runtime behavior. |
</threat_model>
